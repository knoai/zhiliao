# 知了云笔记 · 文档创建与阅读体验重构 PRD

**文档版本**：V1.0  
**编写日期**：2026-04-15  
**功能状态**：设计阶段  
**设计目标**：全面对标语雀（Yuque）的文档创建与阅读交互体验

---

## 一、需求背景

### 1.1 现状痛点

当前「知了云笔记」的文档编辑与阅读体验存在以下明显问题：

| 痛点 | 现状描述 | 用户影响 |
|------|---------|---------|
| 编辑区局促 | 编辑器占满整个屏幕，无舒适留白，长时间写作疲劳感强 | 写作体验差，难以沉浸 |
| 标题位置突兀 | 标题位于顶部导航栏内，与正文割裂 | 不符合主流文档产品心智模型 |
| 阅读体验差 | 编辑态与阅读态无区分，发布后的文档仍像"后台界面" | 知识沉淀感弱，分享意愿低 |
| 大纲导航弱 | 右侧大纲样式简陋，无平滑滚动定位 | 长文档导航困难 |
| 缺少文档仪式感 | 无封面、无文档信息卡片、无专注模式过渡动画 | 产品质感不足 |

### 1.2 设计目标

1. **对标行业标杆**：文档编辑与阅读体验全面对标语雀，降低用户学习成本
2. **提升沉浸感**：通过纸张模拟、专注模式、纯净阅读态打造沉浸式写作与阅读环境
3. **强化信息层级**：标题、正文、大纲、元信息各司其职，界面层级清晰
4. **保留技术资产**：基于现有 Slate 编辑器与 React 技术栈改造，不引入新的富文本引擎

## 二、语雀交互体验拆解

### 2.1 语雀文档编辑器核心特征

#### 2.1.1 整体布局结构（三栏式）

```
┌─────────────────────────────────────────────────────────────────┐
│  [全局顶部导航栏 - 仅保留面包屑与极简操作]                        │
├──────────┬──────────────────────────────┬───────────────────────┤
│          │                              │                       │
│  左侧栏   │       中间编辑/阅读区         │       右侧栏          │
│  文档树   │     （仿 A4 纸张效果）        │      文档大纲          │
│  知识库   │                              │      文档信息          │
│  导航     │                              │                       │
│          │                              │                       │
│  240px   │      自适应，最大 780px        │       220px           │
│          │      最小留白 48px             │                       │
└──────────┴──────────────────────────────┴───────────────────────┘
```

#### 2.1.2 编辑区（Editor Canvas）

- **纸张模拟**：中间区域使用白色背景卡片，带有极淡的阴影 `shadow-sm`，模拟纸张质感
- **舒适留白**：编辑区内容最大宽度 `780px`，两侧自动留白，避免文字顶边
- **标题前置**：文档标题位于编辑区最上方，字号 `32px`，字重 `bold`，与正文融为一体
- **正文排版**：正文字号 `16px`，行高 `1.8`，段落间距 `1em`，符合中文阅读习惯
- **占位符优雅**：标题 placeholder 为 "请输入文档标题..."，正文 placeholder 为 "从这里开始写正文..."

#### 2.1.3 顶部导航栏（Minimal Header）

- **面包屑路径**：显示 `知识库 / 文件夹 / 文档标题`，支持点击回退
- **协作状态**：显示当前在线协作者头像（P2）
- **操作按钮区**：
  - 左侧：「阅读模式」切换按钮
  - 右侧：「发布 / 更新」主按钮、「更多操作」下拉菜单
- **保存状态**：以极细文字显示在右上角，如 "刚刚保存" / "保存中..."

#### 2.1.4 左侧文档树（Doc Tree）

- **知识库维度**：以树形结构展示当前知识库下的文档与文件夹
- **当前高亮**：当前文档以蓝色左侧边框高亮
- **快捷新建**：树顶部有「+ 新建文档」按钮
- **折叠能力**：支持收起左侧栏，收起后显示一个悬浮展开按钮

#### 2.1.5 右侧大纲与信息（Outline & Meta）

- **大纲区域**：
  - 自动提取 H1 ~ H4 标题生成锚点导航
  - 当前阅读/编辑位置通过蓝色高亮指示
  - 点击大纲项平滑滚动至对应位置
  - 无标题时显示 "本文档暂无目录"
- **文档信息卡片（阅读态）**：
  - 作者头像 + 用户名
  - 创建时间、更新时间
  - 字数统计、阅读时长估算

#### 2.1.6 文档底部（Footer）

- **文档信息条**：编辑/阅读态均显示在文档最底部
- 包含：作者、创建时间、最后修改时间、字数
- 视觉弱化处理（`text-gray-400`，`12px`），不喧宾夺主

### 2.2 语雀阅读模式核心特征

#### 2.2.1 阅读态与编辑态的区分

| 维度 | 编辑态 | 阅读态 |
|------|--------|--------|
| 光标 | 显示输入光标 | 隐藏光标 |
| 边框 | 编辑块有 hover 提示 | 无任何编辑提示 |
| 工具栏 | 显示浮动/固定格式工具栏 | 完全隐藏工具栏 |
| 占位符 | 显示 placeholder | 不显示 placeholder |
| 大纲 | 显示，用于编辑定位 | 显示，用于阅读导航 |
| 交互 | 可输入、可格式化 | 仅可阅读、可点击大纲锚点 |

#### 2.2.2 沉浸式阅读体验

- **全屏阅读按钮**：点击后隐藏顶部导航、左侧文档树，仅保留中间正文与右侧大纲
- **优雅的排版**：
  - 标题：H1 `32px` / H2 `24px` / H3 `20px` / H4 `16px`
  - 段落：首行不缩进，段间距 `1em`
  - 引用块：左侧 4px 蓝色竖线 + 浅灰背景
  - 代码块：圆角 6px + 深色背景 + 顶部语言标签 + 复制按钮
  - 列表：列表项与正文左对齐，序号/圆点颜色 `#2563eb`
- **图片交互**：阅读态图片 hover 显示放大按钮，点击可预览

### 2.3 动效与微交互

- **模式切换动画**：编辑态 ↔ 阅读态切换时，工具栏与光标以 `200ms` 淡入淡出过渡
- **左侧栏折叠**：宽度变化使用 `transform` 或 `transition-width`，`300ms ease`
- **大纲高亮切换**：当前锚点高亮项切换有 `150ms` 背景色过渡
- **保存状态提示**：从 "保存中..." 变为 "已保存" 时带有一个小的对勾动画
- **专注模式**：一键进入全屏编辑，隐藏所有 UI 元素（除退出按钮），`300ms` 过渡

## 三、功能范围

### 3.1 MVP 必须实现（P0）

1. **编辑器布局重构**
   - 三栏布局：左侧文档树（240px）、中间编辑区（最大 780px）、右侧大纲（220px）
   - 中间编辑区采用 "纸张" 视觉设计
   - 标题从顶部导航栏移至编辑区正文最上方

2. **编辑态优化**
   - 标题输入框大字号显示，与正文区分
   - 正文 Slate 编辑器排版优化（字间距、行高、段间距）
   - 顶部导航栏极简化为面包屑 + 操作按钮

3. **阅读态实现**
   - 新增「阅读模式」切换开关
   - 阅读态隐藏编辑光标与工具栏
   - 阅读态下 Slate 组件设为 `readOnly`

4. **右侧大纲升级**
   - 大纲样式优化，当前位置高亮
   - 点击平滑滚动定位
   - 阅读态下显示文档信息卡片（作者、时间、字数）

5. **文档底部信息条**
   - 统一显示作者、创建时间、更新时间、字数

### 3.2 后续迭代（P1 / P2）

| 优先级 | 功能 | 说明 |

## 四、页面结构与交互流程

### 4.1 文档编辑页面结构

```tsx
// DocEdit.tsx 结构拆分示意
<DocEditPage>
  ├── <DocEditHeader />          // 顶部极简导航栏
  ├── <div className="flex h-full">
  │     ├── <DocSidebar />       // 左侧文档树（可折叠）
  │     ├── <DocCanvas />        // 中间编辑/阅读区（纸张效果）
  │     │     ├── <DocTitle />   // 大字号标题输入
  │     │     ├── <SlateEditor /> // 正文编辑器
  │     │     └── <DocFooter />  // 底部文档信息
  │     └── <DocOutlinePanel />  // 右侧大纲 + 文档信息
  └── <HistoryDrawer />          // 版本历史抽屉（现有保留）
```

### 4.2 核心交互流程

#### 流程 1：创建新文档

1. 用户点击「新建文档」
2. 进入 `/docs/:id` 编辑页，默认处于 **编辑态**
3. 光标自动聚焦到标题输入框
4. 用户输入标题后按 `Tab` 或点击正文，焦点进入 Slate 编辑器
5. 内容自动保存，右上角提示 "已保存"

#### 流程 2：编辑 ↔ 阅读切换

1. 用户在编辑态点击顶部「阅读模式」按钮
2. 界面切换：
   - 工具栏淡出隐藏
   - 编辑光标消失
   - Slate 设为 `readOnly`
   - 右侧显示「文档信息卡片」
3. 用户点击「编辑文档」按钮可切回编辑态

#### 流程 3：长文档大纲导航

1. 用户在阅读/编辑长文档时滚动页面
2. 右侧大纲实时高亮当前所在章节
3. 用户点击大纲某一项
4. 中间编辑区平滑滚动至对应标题位置（标题顶部留出 `80px` 偏移量）

## 五、视觉规范

### 5.1 色彩规范

| 用途 | 色值 | 说明 |
|------|------|------|
| 编辑区背景 | `#ffffff` | 纯白纸张 |
| 页面背景 | `#f5f6f7` | 语雀风格的浅灰背景 |
| 侧边栏背景 | `#ffffff` | 与编辑区一致，以分隔线区分 |
| 分隔线 | `#e5e7eb` | `border-gray-200` |
| 主题色 | `#2563eb` | 用于高亮、链接、按钮 |
| 正文文字 | `#1f2937` | `text-gray-800` |
| 次要文字 | `#6b7280` | `text-gray-500` |
| 弱化文字 | `#9ca3af` | `text-gray-400`，用于时间、字数 |

### 5.2 字体规范

| 元素 | 字号 | 字重 | 行高 |
|------|------|------|------|
| 文档标题 | `32px` | `600` | `1.3` |
| H1 | `28px` | `600` | `1.4` |
| H2 | `24px` | `600` | `1.4` |
| H3 | `20px` | `600` | `1.4` |
| H4 | `16px` | `600` | `1.4` |
| 正文 | `16px` | `400` | `1.8` |
| 引用/代码 | `14px` | `400` | `1.6` |
| 辅助信息 | `12px` | `400` | `1.5` |

### 5.3 间距规范

| 区域 | 规范 |
|------|------|
| 编辑区内边距 | `48px`（桌面）/ `24px`（移动端） |
| 编辑区最大宽度 | `780px` |
| 段落间距 | `1em` |
| 标题与正文的上下间距 | `0.8em` / `0.4em` |
| 列表项间距 | `0.5em` |
| 大纲项间距 | `6px` 内边距 |

## 六、验收标准

### 6.1 功能验收

- [ ] 文档编辑页呈现三栏布局，左右侧栏可折叠
- [ ] 标题输入框位于编辑区顶部，字号 `32px`，占位符显示正常
- [ ] 编辑区有纸张效果（白色背景、微弱阴影、圆角 `8px`）
- [ ] 正文排版符合中文阅读习惯（行高 `1.8`，段间距 `1em`）
- [ ] 支持「编辑模式」与「阅读模式」一键切换
- [ ] 阅读模式下隐藏工具栏、光标，Slate 不可编辑
- [ ] 右侧大纲可点击跳转，当前章节高亮正确
- [ ] 文档底部显示作者、时间、字数信息
- [ ] 专注模式可用，退出后恢复原有布局

### 6.2 体验验收

- [ ] 页面加载无样式跳动（FOUC）
- [ ] 模式切换、侧边栏折叠有平滑过渡动画
- [ ] 长文档滚动时大纲高亮跟随准确
- [ ] 移动端可正常浏览（侧边栏自动隐藏，编辑区宽度自适应）
- [ ] 现有功能不受影响：自动保存、版本历史、导入导出、发布撤回

### 7.1 参考截图说明

本文档设计全面对标语雀以下页面：
